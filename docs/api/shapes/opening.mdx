## Overview

`Opening` is a cuboid-like solid wrapper used as a subtractive operand in architectural workflows.
It supports kernel-backed placement, outline rendering, freeform conversion, parsed BRep access,
direct subtraction, and the host-oriented `subtractFrom(...)` helper.

## Constructor

```ts
const opening = new Opening(options?: IOpeningOptions);
```

### IOpeningOptions

<ParamField path="ogid" type="string" optional>
  Unique identifier for the opening. Auto-generated when omitted.
</ParamField>

<ParamField path="center" type="Vector3" required>
  Center point of the opening volume.
</ParamField>

<ParamField path="width" type="number" required>
  Width along the X axis.
</ParamField>

<ParamField path="height" type="number" required>
  Height along the Y axis.
</ParamField>

<ParamField path="depth" type="number" required>
  Depth along the Z axis.
</ParamField>

<ParamField path="color" type="number" required>
  Render color as a hex number.
</ParamField>

<ParamField path="fatOutlines" type="boolean" default="false">
  Enables wide outline rendering with `LineSegments2`.
</ParamField>

<ParamField path="outlineWidth" type="number" default="1">
  Outline width used when `fatOutlines` is enabled.
</ParamField>

<ParamField path="translation" type="Vector3" optional>
  Initial placement translation.
</ParamField>

<ParamField path="rotation" type="Vector3" optional>
  Initial placement rotation in radians.
</ParamField>

<ParamField path="scale" type="Vector3" optional>
  Initial placement scale.
</ParamField>

## Methods

### setConfig(), getConfig()

Updates or reads the current opening config.

```ts
opening.setConfig(options: OpeningConfigUpdate): void
const config = opening.getConfig();
```

### getPlacement(), getAnchor()

Returns cloned placement values or the current kernel anchor.

```ts
const placement = opening.getPlacement();
const anchor = opening.getAnchor();
```

### setPlacement(), setTransform(), setTranslation(), setRotation(), setScale()

Placement helpers for moving the opening through the kernel transform path.

```ts
opening.setPlacement({ translation?, rotation?, scale? });
```

### toFreeform()

Converts the opening into a freeform geometry wrapper.

```ts
const freeform = opening.toFreeform(id?);
```

### getBrepData()

Returns the parsed BRep object for the current opening volume.

```ts
const brep = opening.getBrepData();
```

### subtract(), subtractFrom()

Runs a boolean subtraction either with the opening as the host or as the cutting tool.

```ts
const result = opening.subtract(operand, options?);
const cutHost = opening.subtractFrom(host, options?);
```

### generateGeometry(), discardGeometry()

Rebuilds or disposes the current Three.js geometry.

```ts
opening.generateGeometry(): void
opening.discardGeometry(): void
```

## Properties

<ResponseField name="ogid" type="string">

</ResponseField>

## Examples

### Cut a host object

```ts
import { Cuboid, Opening, Vector3 } from "opengeometry";

const wall = new Cuboid({
  center: new Vector3(0, 1.6, 0),
  width: 4.4,
  height: 3.2,
  depth: 0.32,
  color: 0xcbd5e1,
});

const opening = new Opening({
  center: new Vector3(0, 1.35, 0),
  width: 1.4,
  height: 2,
  depth: 0.45,
  color: 0xf97316,
});

const cutWall = opening.subtractFrom(wall, { outline: true });
```

### Reuse the same opening in another position

```ts
opening.setConfig({
  width: 1.8,
  height: 2.1,
  depth: 0.5,
});

opening.setPlacement({
  translation: new Vector3(1.6, 0.1, 0),
  rotation: new Vector3(0, 0, 0),
  scale: new Vector3(1, 1, 1),
});
```
