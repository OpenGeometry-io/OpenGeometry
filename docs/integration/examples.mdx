OpenGeometry ships a compact public example set for `opengeometry`.

The goal is simple: each core demo should be easy to open, easy to read, and small enough to use
as a direct reference when wiring the kernel into your own Three.js scene.

## Core catalog

The source of truth for the public Three.js examples lives in the Three.js adapter package under
`main/opengeometry-three/examples-vite/`.

The current customer-facing catalog includes:

- Primitives: `line`, `arc`, `curve`, `polyline`, `rectangle`
- Shapes: `cuboid`, `cuboid-placement`, `cylinder`, `opening`, `polygon`, `sphere`, `sweep`, `wedge`
- Operations: `boolean-operations`, `polygon-boolean-operations`, `extruded-boolean-operations`, `offset`, `step-export`, `stl-export`, `sweep-path-profile`, `wall-from-offsets`, `editor-modes`

## Hosted demos

You can browse the hosted demo index at [demo.opengeometry.io](https://demo.opengeometry.io/).

The URL paths mirror the `examples-vite/` folder. For example:

- `examples-vite/primitives/line.html` -> `https://demo.opengeometry.io/primitives/line.html`
- `examples-vite/shapes/polygon.html` -> `https://demo.opengeometry.io/shapes/polygon.html`
- `examples-vite/operations/extruded-boolean-operations.html` -> `https://demo.opengeometry.io/operations/extruded-boolean-operations.html`

## Example structure

Each public example page is a standalone HTML file.

The page contract is intentionally simple:

- Static title card markup
- Static control panel markup
- One `#app` mount point
- One inline `<script type="module">`
- Local scene/camera/renderer setup inside that file
- A local `config` object that mirrors the real wrapper API payload
- Local DOM listeners that update the scene directly
- A Stats/FPS panel on the page
- Console logging kept for learning and debugging

The public pages do not rely on a shared example runtime, slug-based loader, or generated example
shell.

## Authoring pattern

The examples follow a simple create-once, update-with-`setConfig(...)` pattern.

```html
<body class="og-example-page">
  <div class="og-badge-title-card">
    <a href="../index.html" class="og-badge-title-link">Back</a>
    <span class="og-badge-title">Cuboid</span>
  </div>

  <aside class="og-controls">
    <!-- static control rows -->
  </aside>

  <div id="app"></div>

  <script type="module">
    import "../src/styles/theme.css";
    import * as THREE from "three";
    import { OrbitControls } from "three/examples/jsm/controls/OrbitControls.js";
    import Stats from "three/examples/jsm/libs/stats.module.js";
    import { OpenGeometry, Cuboid, Vector3 } from "opengeometry";

    const app = document.getElementById("app");
    let cuboid;
    let config;

    function createConfig() {
      return {
        center: new Vector3(0, 0.8, 0),
        width: 2,
        height: 1.6,
        depth: 1.2,
        color: 0x0ea5e9,
        outlineWidth: 2,
      };
    }

    const viewState = {
      outline: true,
    };

    function updateGeometry() {
      cuboid.setConfig({
        center: config.center.clone(),
        width: config.width,
        height: config.height,
        depth: config.depth,
        color: config.color,
        outlineWidth: config.outlineWidth,
      });
      cuboid.outline = viewState.outline;
      console.log("[cuboid] updated", config, viewState);
    }

    async function main() {
      const scene = new THREE.Scene();
      const camera = new THREE.PerspectiveCamera(55, window.innerWidth / window.innerHeight, 0.1, 100);
      const renderer = new THREE.WebGLRenderer({ antialias: true });
      const controls = new OrbitControls(camera, renderer.domElement);
      const stats = new Stats();

      app.replaceChildren(renderer.domElement);
      document.body.appendChild(stats.dom);

      await OpenGeometry.create({ wasmURL: "/opengeometry_bg.wasm" });
      config = createConfig();

      cuboid = new Cuboid({
        center: config.center.clone(),
        width: config.width,
        height: config.height,
        depth: config.depth,
        color: config.color,
        outlineWidth: config.outlineWidth,
      });

      scene.add(cuboid);
      updateGeometry();
    }

    void main();
  </script>
</body>
```

## Package names in local examples vs published apps

In this repository, the example dev/build setup resolves the public package name back to the local
SDK source. In your own application, import from the same published package name:

```ts
import { OpenGeometry, Cuboid, Polygon, Vector3 } from "opengeometry";
```

## Run the examples locally

1. Install dependencies in the repository root.

```bash
npm install
```

2. Start the Three.js example dev server.

```bash
npm --prefix main/opengeometry-three run dev-example-three
```

3. Open the served catalog or a direct page such as `/primitives/line.html` or `/shapes/polygon.html`.

4. Build the catalog for release review when you change examples.

```bash
npm --prefix main/opengeometry-three run build-example-three
```
