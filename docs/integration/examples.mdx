OpenGeometry ships a compact public example set for `@opengeometry/kernel-three`.

The goal is simple: each core demo should be easy to open, easy to read, and small enough to use
as a reference when you are wiring the kernel into your own scene.

## Core catalog

The in-repo source of truth for public Three.js examples is:

- `main/opengeometry-three/examples-vite/index.html`
- `main/opengeometry-three/examples-vite/primitives/*.html`
- `main/opengeometry-three/examples-vite/shapes/*.html`
- `main/opengeometry-three/examples-vite/operations/*.html`
- `main/opengeometry-three/examples-vite/src/styles/theme.css`

The public catalog keeps a curated core set:

- Primitives: `line`, `arc`, `curve`, `polyline`, `rectangle`
- Shapes: `cuboid`, `cylinder`, `opening`, `polygon`, `sphere`, `sweep`, `wedge`
- Operations: `boolean-operations`, `polygon-boolean-operations`, `offset`, `stl-export`, `sweep-path-profile`, `wall-from-offsets`

## Example structure

Each core example is a checked-in standalone HTML page.

The page contract is intentionally simple:

- Static title card in the page body
- Static control panel markup
- One `#app` mount point
- Optional `#stats` only when the page is intentionally a performance demo
- One inline `<script type="module">` with local scene setup and control binding

Core examples do not use a shared page runtime, generated page wrappers, or slug-based loaders.

## Authoring pattern

The current examples follow a single-page pattern: create the object once, keep one local `config`
object, and update geometry through `setConfig(...)`.

```html
<body class="og-example-page">
  <div class="og-badge-title-card">
    <a href="../index.html" class="og-badge-title-link">Back</a>
    <span class="og-badge-title">Cuboid</span>
  </div>

  <aside class="og-controls" data-control-panel>
    <!-- static control rows -->
  </aside>

  <div id="app"></div>

  <script type="module">
    import "../src/styles/theme.css";
    import * as THREE from "three";
    import { OrbitControls } from "three/examples/jsm/controls/OrbitControls.js";
    import { OpenGeometry, Cuboid, Vector3 } from "@opengeometry/kernel-three";

    const app = document.getElementById("app");
    let cuboid;
    const config = {
      width: 1.8,
      height: 1.6,
      depth: 1.2,
      outline: true,
    };

    function setUIControls() {
      // bind existing inputs and update config
    }

    function updateGeometry() {
      cuboid.setConfig({
        center: new Vector3(0, config.height * 0.5, 0),
        width: config.width,
        height: config.height,
        depth: config.depth,
        color: 0x10b981,
      });
      cuboid.outline = config.outline;
    }

    async function main() {
      const scene = new THREE.Scene();
      const camera = new THREE.PerspectiveCamera(55, window.innerWidth / window.innerHeight, 0.1, 4000);
      const renderer = new THREE.WebGLRenderer({ antialias: true });
      const controls = new OrbitControls(camera, renderer.domElement);

      app.replaceChildren(renderer.domElement);
      await OpenGeometry.create({ wasmURL: "/opengeometry_bg.wasm" });

      cuboid = new Cuboid({
        center: new Vector3(0, config.height * 0.5, 0),
        width: config.width,
        height: config.height,
        depth: config.depth,
        color: 0x10b981,
      });

      scene.add(cuboid);
      updateGeometry();
      setUIControls();
    }

    void main();
  </script>
</body>
```

In this repository, local example authoring uses the `@og-three` alias inside the Vite config. When
you copy the pattern into an application or a separate example repo, import from the published
package name `@opengeometry/kernel-three`.

## Shared styling

The shared example UI skin lives in:

- `main/opengeometry-three/examples-vite/src/styles/theme.css`

That stylesheet is used by the public example pages for:

- The compact title card
- The compact right-side control panel
- Shared input, toggle, textarea, and button styling

The goal is utility UI, not showcase UI. Control panels should feel like production tooling.

## Extra requests

The public catalog stays intentionally small. If you want to see additional example coverage,
surface the request through GitHub or Discord and keep the request tied to a concrete workflow.

## Run the core examples locally

1. Install dependencies in the repository root.

```bash
npm install
```

2. Start the Three.js example dev server.

```bash
npm --prefix main/opengeometry-three run dev-example-three
```

3. Open the served catalog or a direct page such as `/primitives/line.html` or `/operations/stl-export.html`.

4. Build the catalog for production review when you change examples.

```bash
npm --prefix main/opengeometry-three run build-example-three
```
