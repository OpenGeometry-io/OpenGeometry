---

OpenGeometry exposes kernel-backed Three.js wrappers from the public `opengeometry` package. This is
the main browser-side integration layer for web CAD, AEC/BIM, and geometry-heavy Three.js tools.

## Installation

```ts
import {
  OpenGeometry,
  Line,
  Arc,
  Curve,
  Polyline,
  Rectangle,
  Polygon,
  Solid,
  Cuboid,
  Cylinder,
  Sphere,
  Sweep,
  Wedge,
  Opening,
  Vector3,
} from "opengeometry";
```

## Initialization

Initialize the OpenGeometry runtime once before you construct `Vector3` values or wrappers:

```ts
await OpenGeometry.create({
  wasmURL: "/opengeometry_bg.wasm",
});
```

## Wrapper model

### Primitive wrappers

`Line`, `Arc`, `Curve`, `Polyline`, and `Rectangle` are line-based wrappers. They extend
`THREE.Line` and use `setConfig(...)` plus placement helpers such as `setPlacement(...)`,
`setTranslation(...)`, `setRotation(...)`, and `setScale(...)`.

### Shape wrappers

`Polygon`, `Cuboid`, `Cylinder`, `Sphere`, `Sweep`, `Wedge`, and `Opening` are mesh-backed wrappers.
They extend `THREE.Mesh`, render directly in a Three.js scene, and expose boolean-friendly BRep
accessors or subtract helpers.

### Solid wrapper

`Solid` is the generic BRep-backed mesh wrapper returned by workflows like:

```ts
const wall = polygon.extrude(2.8);
```

## Common patterns

### Create once, then update with setConfig

```ts
const cuboid = new Cuboid({
  center: new Vector3(0, 0.8, 0),
  width: 1.5,
  height: 1.6,
  depth: 1.2,
  color: 0x10b981,
});

cuboid.setConfig({
  width: 2,
  height: 2.2,
  depth: 1.4,
});
```

### Use placement helpers for modeling transforms

For wrapper geometry, prefer the wrapper placement APIs when you want the kernel-backed geometry to
move with the object state:

```ts
cuboid.setPlacement({
  translation: new Vector3(2, 0, 0),
  rotation: new Vector3(0, Math.PI / 4, 0),
  scale: new Vector3(1, 1, 1),
});
```

### Use Three.js scene APIs normally

The wrappers still behave like normal Three.js objects:

```ts
scene.add(cuboid);
scene.add(line);
```

## Examples by wrapper family

### Polygon and extrusion

```ts
const polygon = new Polygon({
  vertices: [
    new Vector3(-1, 0, -0.2),
    new Vector3(1, 0, -0.2),
    new Vector3(1, 0, 0.2),
    new Vector3(-1, 0, 0.2),
  ],
  color: 0x2563eb,
});

polygon.outline = true;

const wall = polygon.extrude(2.8);
wall.outline = true;
scene.add(wall);
```

### Cuboid

```ts
const cuboid = new Cuboid({
  center: new Vector3(0, 0.85, 0),
  width: 1.9,
  height: 1.7,
  depth: 1.2,
  color: 0x2563eb,
});
```

### Cylinder

```ts
const cylinder = new Cylinder({
  center: new Vector3(0, 1, 0),
  radius: 0.9,
  height: 2,
  segments: 48,
  angle: 2 * Math.PI,
  color: 0xf97316,
});
```

### Sphere

```ts
const sphere = new Sphere({
  center: new Vector3(0, 1.1, 0),
  radius: 1.1,
  widthSegments: 28,
  heightSegments: 18,
  color: 0x0891b2,
});
```

### Sweep

```ts
const sweep = new Sweep({
  path: [
    new Vector3(-1.4, 0, -1.6),
    new Vector3(-0.5, 0.6, -0.5),
    new Vector3(0.4, 1.3, 0.8),
    new Vector3(1.2, 2.4, 2.1),
  ],
  profile: [
    new Vector3(-0.4, 0, -0.25),
    new Vector3(0.4, 0, -0.25),
    new Vector3(0.4, 0, 0.25),
    new Vector3(-0.4, 0, 0.25),
  ],
  color: 0x3b82f6,
  capStart: true,
  capEnd: true,
});
```

## Booleans

You can use either the standalone boolean helpers or the shape-level convenience methods. For AEC
workflows, `Opening.subtractFrom(...)` is usually the clearest host-cutting path.

```ts
const opening = new Opening({
  center: new Vector3(0, 1.35, 0),
  width: 1.4,
  height: 2,
  depth: 0.45,
  color: 0xf97316,
});

const cutWall = opening.subtractFrom(cuboid, {
  outline: true,
  fatOutlines: true,
  outlineWidth: 2,
});
```

## BRep and freeform access

Different wrappers expose different BRep helpers. The main patterns are:

- Parsed BRep object: `getBrep()` or `getBrepData()`
- Serialized BRep JSON: `getBrepSerialized()` or `getLocalBrepSerialized()`
- Freeform conversion: `toFreeform(id?)`

`Solid` exposes the broadest set of BRep helpers because it is the generic BRep-backed wrapper.
